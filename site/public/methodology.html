SOXX Regime Monitor — Methodology

This page documents the signal construction, state machine rules, and backtest assumptions underlying the dashboard.

1. Overview

The monitor tracks a return-decomposition signal on SOXX (iShares Semiconductor ETF). Each daily return is split into:

  • Overnight (on) = log(opent / closet-1) — retail/news-driven gap
  • Intraday (id) = log(closet / opent) — institutional intraday flow

A 20-session rolling sum of each gives on20 and id20. Persistent negative id20 while on20 stays positive signals distribution (institutions selling into overnight strength). This divergence precedes drawdowns.

2. Defined Terms

All metrics used in this dashboard, in plain English.

Daily Return Components

on overnight return
The price change from the previous day's close to today's open — what happened while the market was shut.
Formula: log(opent / closet−1). Driven by news, macro data, and retail order flow.
id intraday return
The price change from today's open to today's close — what happened during the trading session.
Formula: log(closet / opent). Dominated by institutional order flow.
ret total daily return
The full day's log-return: previous close to current close. Equal to on + id.

Rolling Stream Signals

id20 20-day intraday stream
The sum of the last 20 intraday returns. This is the core decision signal.
When negative, institutions have been net sellers during the trading day over the past month, even if the stock is up. This is the distribution signal that arms the monitor.
on20 20-day overnight stream
The sum of the last 20 overnight returns. Captures the retail/news-driven bid.
When on20 stays positive while id20 turns negative, it means institutions are quietly selling into overnight gaps — the defining divergence pattern.
ret20 20-day total return
The cumulative log-return over the last 20 sessions. Measures the current 1-month trend.
Used in the arm condition (distribution is only flagged when the trend is still up) and the accumulation overlay (price falling with id20 positive).

Volatility Metrics

rv20 20-day realized volatility
The annualized standard deviation of daily log-returns over the past 20 sessions.
Used to scale position size: full exposure only when volatility is below ~50% annualized. High volatility = smaller suggested size.
rv10 10-day realized volatility
Same as rv20 but over 10 sessions — a shorter-term volatility gauge, more sensitive to recent spikes.
iv30 30-day implied volatility
The market's expectation of volatility over the next 30 days, derived from options prices. Entered manually from options market data.
vrp variance risk premium
Implied vol minus realized vol: iv30 − rv20.
Positive = options are expensive relative to recent moves (normal; sellers get paid a premium). Negative = options are cheap, which often happens during sharp sell-offs when realized vol spikes above implied. Negative VRP suggests hedging with puts is cheap.

Trend & Momentum Indicators

ma20 / ma50 / ma200 moving averages
Simple moving averages of closing prices over 20, 50, and 200 sessions respectively.
ma20 is the re-entry line: price must recover above it (for 2 consecutive sessions with positive id20) to trigger a trend-reclaim re-entry from EXIT state.
rsi14 relative strength index
A momentum oscillator (0–100) measuring the speed and magnitude of recent price changes over 14 sessions.
Above 70 = overbought; below 30 = oversold. Used as a supplementary context indicator — not a direct input to the state machine.
dist20 sessions since MA20 cross
How many trading sessions have elapsed since price last crossed above or below the 20-day moving average.
A large number means price has been persistently above or below the line — indicating a more entrenched trend.
ar1 return autocorrelation
The first-order autocorrelation of daily returns over the last 20 sessions.
Positive = returns are trending (today up → tomorrow likely up). Negative = mean-reverting (today up → tomorrow likely down). Extreme negative values can signal choppy, whipsaw conditions.

Sentiment & Risk Indicators

turb turbulence score
A statistical measure of how unusual today's return is relative to the recent 20-day distribution, expressed as a Z-score.
A score above 3 means today's move is more than 3 standard deviations from recent average — a tail event. The Jun 5, 2026 crash (−10.4%) scored ~3.7.
pc_oi put/call open interest ratio
The total open interest of put options divided by call options on SOXX (entered manually from options market data).
High ratio = more puts outstanding than calls = bearish hedging in the market. Useful as a sentiment cross-check; not a direct state machine input.
iv30_asof
The date the IV30 reading was last manually updated. Shown to flag when the options data may be stale.

4. State Machine

The system operates in five states:

StateColourMeaning
WARMUP■ GrayFirst 20 sessions of year — insufficient history
RISK_ON■ GreenFully invested; no distribution signal
MONITOR■ AmberDistribution detected; monitoring for exit trigger
EXIT■ RedFlat / defensive; awaiting re-entry conditions
ACCUM■ BlueOverlay: price weak but intraday flow positive — stay invested

4.1 Arm Condition (RISK_ON → MONITOR)

Either of:

  • DIV arm: id20 < 0 AND ret20 > +2% (distribution while trend still up)
  • ABS arm: id20 < −3% (extreme intraday selling regardless of trend)

Arm condition must be true at open; does not fire during ACCUM.

4.2 Exit Trigger (MONITOR → EXIT)

Exec-into-strength: idt > +1% OR rett > +1.5% while armed condition is still true.

Same-day arm+exit: if arm and exit conditions both trigger on the same session, exit fires immediately (MONITOR band shown for that day only).

Escape valve: after 3 sessions in MONITOR with no strength day AND close < MA20, exit anyway.

4.3 Disarm (MONITOR → RISK_ON, no trade)

If the arm condition clears (reverses) before an exit executes → cancel, no trade. This takes priority over exec-into-strength on the day the condition clears.

4.4 Accumulation Override (RISK_ON + ACCUM overlay)

id20 > +2% AND ret20 < −2% — price falling but institutions buying intraday. Remain invested; arm condition does not fire during ACCUM.

4.5 Re-entry (EXIT → RISK_ON)

Any of:

  • Disarm: arm condition clears while in EXIT → immediate re-entry
  • Accumulation flip: acc condition triggers while in EXIT → re-entry
  • MA20 recovery: 2 consecutive sessions above MA20 while id20 > 0

5. Position Sizing

Suggested size is scaled by realised volatility: size = min(1, 0.50 / rv20) when RISK_ON. Full position only when vol < 50% annualised. EXIT = 0%.

6. Backtest Assumptions

  • YTD start = first trading session of Jan 2026; WARMUP = first 20 sessions
  • State machine resets each Jan 1
  • Equity curve: 1× strategy (exit → cash earning 0%), vs 1× B&H
  • No slippage, commissions, or borrowing cost modelled
  • Historical window: 420 calendar days for rolling computations

7. Data Sources

OHLCV data: yfinance → stooq → stockanalysis (fallback chain). Updated daily via GitHub Actions at 22:15 UTC on weekdays. IV30 and P/C OI are entered manually in data/manual.json.
